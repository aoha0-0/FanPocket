# FanPocket (ファンポケット)

> **「忙しい推し活に、確実な安心感と使いやすさを。」**  
> 「あとでやろう」「あとで買おう」を後悔にしない、URL保存から行動へ直結する推し活特化型スケジュール管理アプリ。

---

## 💡 サービス概要

FanPocketは、公式X（旧Twitter）やWebサイトで溢れるチケット先行・グッズ販売などの情報を、URLごと手間なく保存し、タイミングを逃さず行動につなげるためのスケジュール管理アプリです。

情報が混在して埋もれがちな既存のカレンダーアプリや、タスクの消化を急かされるビジネスライクなToDoツールとは異なり、**「推し活の情報だけに集中できるノイズレスさ」**を大切にした、ジャンルを問わず使いやすく見やすい空間を提供します。

### 🎯 解決したい課題
* **後回しによる機会損失の防止:** 「あとで申し込もう」「あとで買おう」と思ったまま忘れてしまい、受注販売や数量限定グッズを買い逃す悔しい体験（実体験）をなくします。
* **手入力ミスの解消:** カレンダーアプリへの手動入力で発生しがちな「日付の登録ミス」を、URLベースの管理と入力補助機能によって防ぎます。
* **心理的ストレスの軽減:** 過剰な通知や数値評価に縛られず、ユーザーが推しの予定と静かに、安心して向き合える状態を作ります。

## 💜 開発背景（ストーリー）

### きっかけとなった実体験
私自身、多ジャンルの推し活を日常的に楽しんでいます。情報は公式XやLINE、各種サイトから収集していましたが、情報を見つけたその場では「あとで申し込もう」「あとで買おう」と後回しにしてしまうことが多々ありました。
ある時、Xのポストで大好きなグッズの受注販売期間が過ぎていたことを知り、非常に大きなショックを受けました。「発売を知っていたのに、情報を掴んでいたのに、忘れていたせいで買えなかった」という悔しさは、今でも鮮明に覚えています。
この「忘れてしまうもったいなさ」を解消し、誰もが後悔のない推し活を安心して楽しめるようにしたい、と考えたことがこのアプリの原点です。

### 既存サービスへの違和感と差別化
カレンダーや一般的なToDoアプリ（TimeTree, Todoist, Google Calendar等）での管理も試みましたが、以下の課題がありました。
1. **情報の埋没:** 日常の予定や仕事のタスクと推し活の情報が混在し、重要な締め切りが埋もれてしまう。
2. **登録の手間による後回し:** 登録時にタイトルや日時を細かく整理・手入力する必要があり、その手間のせいで結局「あとでやろう」と後回しにしてしまう。また、手入力による日付の登録ミスも発生しやすい。

**【FanPocketの差別化ポイント】** 見つけた情報を**「その場でURLごと1タップ感覚で保存」**でき、元の情報（ソース）にいつでも再確認・遷移できる動線を確保。さらに、未対応の予定に絞ったスマートな通知により、ユーザーの「あとで」を確実に行動へと繋げます。

## 💎 独自のこだわり・技術的工夫

### インフラ：完全無料（コストゼロ）で高可用性を実現するアーキテクチャ
個人開発における「運用コスト（無料枠の制限）」と「ユーザー体験（コールドスタートや通知の遅延）」のトレードオフを、技術的なアプローチで解決しています。

* **Serverlessスタックによるリソース最適化:**
  リレーショナルデータにはPostgreSQL、非同期処理用のデータストアには **Upstash (Redis)** を採用。いずれも従量課金かつ無料枠が非常に高機能なサーバーレス技術を組み合わせることで、**インフラ費用を完全にゼロ**に抑えつつ、本番環境に必要なパフォーマンスを確保しています。
* **GitHub Actionsによる「プレウォーム（予熱）」ハック:**
  Renderの無料プランにおける最大の弱点である「一定時間未アクセスによるスリープ」を解決するため、GitHub Actionsを利用。
  ユーザーのアクティブ時間や、採用担当者様がアクセスする時間帯の直前に、自動で軽量なヘルスチェックリクエストを送信してインスタンスを「事前に起こしておく（プレウォーム）」仕組みを構築。有料プランへ移行することなく、コールドスタートによる初速の遅さを技術的工夫でカバーしています。

### UI/UX：ノイズレスなデザインと「片手操作」への徹底的な執着
情報収集から保存までの流れの多くが「スマホ」で行われるため、「インフラ費用は1円もかけないが、スマホでの操作性は絶対に妥協しない」という方針を貫いています。

* **モバイル最適化 UI/UX:**
  * **ボトムナビゲーションの導入:** スマホを片手で持った際、親指が自然に届く画面下部（親指タッチエリア）に、主要アクション（追加、一覧、検索）を配置。画面上部に手を伸ばすストレスを無くし、日常の移動中などでもサクサク操作できるUIを実現しています。
  * **PWA (Progressive Web App) 化:** ホーム画面へアイコンを追加し、ブラウザの検索バーを非表示にすることで、ネイティブアプリのような起動速度とフル画面の操作感を**「追加のインフラコスト完全ゼロ」**で実現します。
* **「飾らない、情報に集中できる」場所の提供:**
  効率性を追い求める無機質なタスク管理アプリとは対照的に、複数ジャンルの推し活をしていても情報が混ざらず、すっきりと整理できるクリーンな世界観を徹底。過剰な通知機能や派手な配色を排除し、Tailwind CSSとDaisyUIをカスタマイズして、ユーザーが「開始・締切のタイミング」をひと目で把握できるトーン＆マナーを実装しました。

### Rails + Hotwireによる「高コスパ・高UX」の追求
フロントエンドとバックエンドを完全に分離（SPA化）するアプローチは、インフラの複雑化やホスティングコストの増加に繋がります。本アプリでは、Rails 7.1の標準機能である **Hotwire (Turbo / Stimulus)** をフル活用しています。

* **単一リポジトリでの運用コスト削減:** インフラ構成をシンプルに保ち、デプロイや環境構築の手間を最小限化。
* **スマホでも途切れないサクサク感:** ページ全体をリロードしない画面遷移（Turbo）や、保存完了・ステータス切り替えを瞬時に行うUIを、追加のサーバー費用なしで実現。

## 🧱 機能一覧

### 🐣 MVP（最小限の実行可能プロダクト）
* **ユーザー認証機能**（Deviseによるサインアップ/ログイン/ログアウト）
* **URL保存機能**（見つけた情報をその場で保存、URLへの1タップ遷移）
* **スケジュール管理機能**（開始日時・締切日時の任意登録、メモ機能）
* **一覧・詳細表示機能**（日付順での並び替え、ステータス：完了/未対応 の切り替え）
* **リマインドメール通知機能**（未対応タスクに対し、3日前・前日20時・当日7時に確実な非同期配信）

### 🚀 本リリース予定
* **タイトル自動取得機能**（MetaInspectorによるURL解析・手動入力への安全なフォールバック付き）
* **日付候補の抽出機能**（URL内の日付を自動抽出し、スマホでの手入力を極限まで減らす補助機能）
* **UI・体験のさらなる改善**
  * タグ機能の追加、状態（未対応/完了）に応じた色分け表示。
  * 期限切れの項目を下部に自動移動させることで、今対応すべきタスクへの視覚的集中を促す。
* **前述のモバイル最適化（ボトムナビゲーションの実装 / PWA化）**
* **通知の多角化:** Web Push通知（ブラウザ通知）、LINE Messaging API連携（需要・コストに応じて検討）

### データの蓄積による「思い出の資産化」
単なるタスク管理ツールは用が済めば削除されますが、FanPocketは使い続けることで、過去に保存したイベントや応募履歴が「参加した記録」や「推し活の思い出」として蓄積されます。カレンダー形式での情緒的な振り返り機能などを通じ、愛着を持って長く使い続けられるサービスを目指します。
